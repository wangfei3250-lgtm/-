## 核心组件

### 1. 编剧导板 (ScriptRunner)

负责编剧流程的主控制器：

```
初始化
  ↓
加载配置和智能体
  ↓
循环执行每个智能体
  │
  ├─ 检索知识库
  ├─ 生成提示词
  ├─ 调用 AI 模型
  ├─ 解析输出
  ├─ 存储结果
  └─ 准备下一个智能体的输入
  ↓
汇总结果
  ↓
生成输出文件
```

### 2. 智能体管理 (AgentManager)

管理 11 个编剧智能体的配置和执行：

```
agents.json
  ↓
解析配置
  ↓
为每个智能体构建：
  - 系统提示词
  - 职责和关注点
  - 交接要求
  - 质量标准
```

### 3. 知识库系统 (KnowledgeBase)

```
用户导入资料
  ↓
文件预处理（支持 txt, md, docx, pdf 等）
  ↓
文本分块和向量化
  ↓
存储到 SQLite + 向量索引
  ↓
实时搜索和检索
```

### 4. AI 模型提供商 (Providers)

支持多种 AI 服务：

```
Provider 接口
├─ OfflineProvider: 本地规则生成
├─ OpenAIProvider: OpenAI API 兼容
└─ DeepSeekProvider: DeepSeek API
```

### 5. 输出生成 (OutputGenerator)

```
编剧讨论结果
  ↓
Markdown 格式化
  ↓
Word 文档生成 (python-docx)
  ↓
PDF 导出（可选）
```

## 数据流

### 典型编剧流程

```
1. 用户输入故事简报
   {
     "title": "故事名",
     "genre": "类型",
     "premise": "前提"
   }

2. 总编剧处理
   输入: 简报 + 系统提示
   输出: 创作北极星、核心卖点

3. 故事架构师处理
   输入: 总编剧输出 + 简报
   输出: 三幕结构、关键转折

4. 后续智能体依次处理...
   
5. 统筹编辑综合汇总
   输入: 所有智能体输出
   输出: 完整工作清单

6. 生成最终输出
   Markdown 报告 + Word 剧本
```

### 人工干预流程

```
编剧进行中
  ↓
用户在网页 @某智能体
  ↓
意见保存到内存
  ↓
影响后续智能体的处理
  ↓
下一次运行调整策略
```

## 关键设计模式

### 1. 责任链模式 (Chain of Responsibility)

每个智能体处理完成后自动传递给下一个智能体。

### 2. 策略模式 (Strategy)

支持不同的 AI 提供商（离线、OpenAI、DeepSeek）通过统一接口。

### 3. 模板方法模式 (Template Method)

所有智能体遵循相同的执行框架。

### 4. 观察者模式 (Observer)

Web UI 通过 WebSocket 实时监听编剧进度。

## 扩展性考虑

### 添加新智能体

1. 在 `agents.json` 中定义新角色
2. 提供清晰的职责和关注点
3. 系统会自动集成

### 添加新 AI 提供商

1. 继承 `Provider` 基类
2. 实现 `generate()` 方法
3. 注册到 `ProviderFactory`

### 自定义输出格式

1. 实现 `OutputGenerator` 子类
2. 支持新的文件格式
3. 在 API 中暴露导出选项
